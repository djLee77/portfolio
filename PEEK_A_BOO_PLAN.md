# 🎭 Peek-a-boo Animation Plan (ThinkSection)

Claude Code, 당신은 포트폴리오의 첫 번째 섹션인 `ThinkSection.tsx`에 사용자 피드백을 반영하여 아주 유쾌하게 텍스트와 이미지가 상호작용하는 "까꿍(Peek-a-boo)" 애니메이션 시퀀스를 추가해야 합니다.

## ✅ 시퀀스 기획 (순서대로 구현할 것)

1. **텍스트 등장 (기존 유지)**:
   - 4줄의 텍스트("나는..." 등)가 뷰포트에 도달했을 때 밑에서 솟아오르는 `staggerChildren` 스프링 애니메이션은 이미 훌륭하니 그대로 둡니다.

2. **텍스트 그룹 우측 비켜서기 (Shift Right)**:
   - 텍스트 전체를 감싸고 있는 `motion.div` 래퍼(Wrapper) 쪽에 별도의 `whileInView` 모션을 걸어주세요.
   - **조건**: 텍스트가 모두 튀어오르는 약 1.2초 뒤(`delay: 1.2`)에, 전체 텍스트 덩어리가 우측으로 살짝 이동(`x: '15vw'`)하며 빈자리를 내어주어야 합니다. 좌우로 긁힐 때마다 반복되지 않게 무조건 `viewport={{ once: true }}`로 막아야 합니다.

3. **내 캐릭터 이미지 등장 (Peek-a-boo Pop-up)**:
   - 텍스트 덩어리가 비켜주는(1.2초) 타이밍에 맞춰 **1.3초**(`delay: 1.3`)에 좌측 숨겨진 곳에서 이미지가 튀어나와야 합니다.
   - `/character.png` 경로를 가진 `motion.img` 태그를 `div` 안쪽에 새로 만들고 `absolute`로 왼쪽 밑 끄트머리에 구겨 넣으세요. (`bottom: -5vh, left: 0`)
   - **애니메이션**: `initial={{ x: '-100%', rotate: -25, opacity: 0 }}` 상태로 대기하다가, 특정 시간이 되면 `whileInView={{ x: '-5%', rotate: 0, opacity: 1 }}`로 튕겨 나옵니다. (`type: 'spring', damping: 12` 텐션 강력 추천)
   - 이 이미지 또한 텍스트와 마찬가지로 무조건 `once: true`로 단 한 번만 실행되어야 촌스럽지 않습니다.

## ⚠️ 주의사항
캐릭터가 좌측 바깥에서 들어올 때 삐져나가서 가로 스크롤바가 생기지 않도록 `<section>` 부모 최상단에 반드시 `overflow: 'hidden'`을 추가해야 완벽한 밀실 까꿍 모션이 완성됩니다.
